## 🚀 Requisitos Previos

Antes de ejecutar la aplicación, asegúrese de tener instalado en su equipo:
* **.NET SDK** versión `8.0` o superior. Puede verificarlo ejecutando:
  ```bash
  dotnet --version
  ```
* **PostgreSQL** versión `14` o superior en ejecución local (usuario `postgres`).
* Un navegador web moderno (Edge, Chrome, Firefox, etc.).

---

## ⚙️ Configuración de la Base de Datos

1. Asegúrese de tener PostgreSQL corriendo y una base de datos llamada `mi_tienda` creada (o deje que EF Core la cree con las migraciones).
2. Verifique la cadena de conexión en `appsettings.json`:
   ```json
   "ConnectionStrings": {
     "DefaultConnection": "Host=localhost;Database=mi_tienda;Username=postgres;Password=<SU_CONTRASEÑA>"
   }
   ```
3. Instale la herramienta global de EF Core (solo la primera vez):
   ```bash
   dotnet tool install --global dotnet-ef
   ```
4. Ejecute las migraciones para crear las tablas:
   ```bash
   dotnet ef database update
   ```

---

## 🛠️ Instrucciones de Ejecución

1. Abra una terminal en el directorio del proyecto (donde se encuentra `MiPrimerMVC.csproj`).
2. Restaure las dependencias e inicie el servidor de desarrollo:
   ```bash
   dotnet run --launch-profile "http"
   ```
3. La aplicación estará disponible en:
   * [http://localhost:5087](http://localhost:5087)
4. Acceda al módulo de gestión de productos en:
   * [http://localhost:5087/Producto](http://localhost:5087/Producto)

*Nota: Para habilitar Hot Reload durante el desarrollo:*
```bash
dotnet watch
```

## ✨ Características y Directrices Implementadas

### 1. Modelo `Producto` con Data Annotations
El modelo de datos `Producto.cs` ubicado en `Models/` incluye validaciones enriquecidas utilizando decoradores (`DataAnnotations`):
* **Id**: Clave primaria auto-incremental (gestionada por PostgreSQL con `GENERATED BY DEFAULT AS IDENTITY`).
* **Nombre**: Obligatorio, longitud entre 3 y 100 caracteres.
* **Precio**: Obligatorio, rango válido de $0.01 a $10,000.00.
* **Descripción**: Opcional, longitud máxima de 250 caracteres.
* **Stock**: Obligatorio, valor no negativo hasta un máximo de 1000 unidades.

### 2. Persistencia con Entity Framework Core + PostgreSQL
* El `ApplicationDbContext` (en `Data/`) gestiona la sesión con la base de datos PostgreSQL.
* El `ProductoController` usa **consultas asíncronas** (`async/await`) con EF Core, que internamente utiliza **consultas parametrizadas** (equivalente a prepared statements), previniendo inyección SQL.

### 3. Controlador `ProductoController`
Implementa la lógica CRUD con las siguientes acciones en `Controllers/ProductoController.cs`:
* **Index (GET)**: Recupera la lista de productos desde la base de datos.
* **Create (GET)**: Retorna el formulario de creación limpio.
* **Create (POST)**: Valida el modelo y persiste el nuevo producto en PostgreSQL.
* **Delete (GET)**: Recupera el producto específico por su `id`.
* **Delete (POST)**: Elimina el producto de la base de datos y redirige al listado.

### 4. Vistas Razor y Validación del Lado del Cliente / Servidor
Las vistas en `Views/Producto/` fueron estructuradas con diseño responsivo usando **Bootstrap 5**:
* **`Index.cshtml`**: Tabla responsiva con badges de stock (rojo = sin stock, amarillo = bajo, celeste = óptimo).
* **`Create.cshtml`**: Formulario con validación dual (servidor vía `ModelState.IsValid` + cliente vía jQuery Validation).
* **`Delete.cshtml`**: Vista de confirmación de eliminación con advertencias visuales.

### 5. Seguridad CSRF y Anti-Forgery Tokens
Para prevenir ataques CSRF:
* Atributo `[ValidateAntiForgeryToken]` en las acciones POST de `Create` y `Delete`.
* Token inyectado en formularios Razor mediante `@Html.AntiForgeryToken()`.

## 🔒 Controles de Seguridad OWASP Implementados

| Control | Implementación |
|---------|---------------|
| **Prevención XSS** | Razor codifica automáticamente toda salida con `@variable` (HTML Encoding) |
| **Token CSRF** | `[ValidateAntiForgeryToken]` + `@Html.AntiForgeryToken()` en todos los formularios |
| **Consultas Seguras** | EF Core usa consultas parametrizadas internamente (prevención de SQL Injection) |
